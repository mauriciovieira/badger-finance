[bearbeiten] 3 BADGER finance Version 1.0 beta 2 (2006-08-26)

  • Added: Hint that there should be no Apache and MySQL Server upon installing with the installer (FS#84).
  • Fixed: Registry entry stays after uninstalling (FS#91).
  • Fixed: Current balance not correct at account overview (FS#92).
  • Fixed: Sorting the categories in the selection dropdown now is alphabetic (FS#94).
  • Fixed: User setting "Auto-insert recurring transactions" is now possible to uncheck (FS#95).
  • Fixed: Our JS form validation is now compatible with WhatWG web forms specification (made some forms unusable in Opera 9) (FS#96).
  • Fixed: Wrong recognition of amounts > 1000 in Sparkasse parser (FS#97).
  • Fixed: Wrong capitalization of table names (led to several error messages on Unix systems, e. g. when creating new accounts) (FS#98).
  • Fixed: Double entries of recurring transactions (FS#100).
  • Fixed: Cannot leave out end date on editing recurring transaction (FS#102).
  • Fixed: Wrong return page on editing recurring transactions out of the list of recurring transactions (FS#104).
  • Fixed: Session timeout now working (FS#107).
  • Fixed: Initial login screen German (FS#109).
  • Fixed: Wrong error message "Date in the future" when displaying statistics (FS#110).
  • Fixed: Internationalization of statistics (FS#111).
  • Added: User setting of how long to pre-calculate future recurring transactions (FS#112).
  • Changed: Parser recognition. Now a new parser is recognized just by copying the parser to the appropriate directory (FS#115).
  • Fixed: Layout problems in forecast in English (FS#118).
  • Fixed: The currency is now written next to the forecast results (FS#122).
  • Added: American date format (mm/dd/yy) (FS#123).
  • Added: Parser for Bank of America (FS#125).
  • Fixed: Internal link from recurring transactions to it's single instances (useful for further improvement) (FS#129).
  • Added: Color of positive and negative amounts in the Grid differ (FS#131).
  • Fixed: CSS styles of calendar hardcoded (FS#133).
  • Fixed: Login URL contained two consecutive slashes (FS#135).
  • Fixed: Cannot use Full Year / Month selection box if date format is mm/dd/yy (FS#139).
  • Fixed: User setting "Start page" could be empty (FS#142).
  • Added: Method for updating previous BADGER finance installations.
  • Added: Backups from previous versions of BADGER finance can now be imported.
  • Added: Release Notes entry in help menu.
  • Added: New welcome page.
  • Added: Non-Standard templates do not need to implement all template files.
  • Fixed: Template changes are carried out immediatly on User settings page.
  • Added: Link to badger-finance.org on every page.
  • Added: Improved user guidance on update and backup restore.
  • Added: Developed tool to compare databases.
  • Added: Primary keys to all database tables.
  • Added: Update installer.
  • Fixed: Timezone hack for several PHP version >= 5.1.0.
  • Fixed: Changing default password in Internet Explorer didn't work.
